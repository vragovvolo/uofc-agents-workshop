What is an agent, actually?

agent = LLM + tools + a loop
while True:
    response = llm(messages, tools=[run_sql, ...])   # LLM decides
    if not response.tool_calls:                      # nothing left to do?
        return response.content                      # → final answer
    for call in response.tool_calls:                 # otherwise: act,
        messages.append(execute(call))               #   observe, repeat
  • The LLM chooses which tool to call, with what arguments, based on what it has seen so far
  • Errors go back into the loop — the agent reads them and fixes itself
  • Everything else (multi-agent, RAG, "deep research") is variations on this loop

④+⑤ Specialists and the supervisor

Why specialists?

  • Focused prompt + small table scope = better SQL, fewer hallucinations
  • Independently testable, swappable, ownable
  • Same reason companies have teams

Why a supervisor?

  • To the supervisor, each agent is just a tool
  • It routes, asks follow-ups, synthesizes
  • Cross-domain answers no single agent could give
